Overview

  • Git and GitHub are not the same. Simply speaking, Git is a tool that is used to manage source code, whilst GitHub is a technology where we push our codes from the computer to. Using GitHub is not compulsory when it comes to Git, but this does not apply for the other way around.
  • When working in a group, using Git is necessary. It can audit the changes, keep track on the progress and store the library. In short, Git makes collaboration easier and helps us with our work as Programmer. For GitHub, it is an open source, somewhat similar to Facebook. It works as a portforlio since all of our projects can be stored here for employers and other users to see, and also shows our contributions from the pull requests.
  • All in all, if you are interested in becoming a programmer, start learning how to use Git and GitHub.

    Basic Git Commands

    Open new file and name it “index.html”, type in some basic lines, then press Ctrl + S

    Open "Terminal",type in “git add [name of the file]”. If you want to update everything, type in “git add .”

    Next, type “git commit –m ‘[write in the message]’”

    Finally type “git push” to push your codes into your GitHub, the result when it’s done will be like this:

    Branching

    Type git branch to check on your branches

    Type in “git branch + [new branch name]”

    Type in “git checkout + [branch name]” to switch to the branch that you want

    Use git merge to merge branches together
